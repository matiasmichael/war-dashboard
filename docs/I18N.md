# Internationalization (i18n) — Middle East Pulse

Documentation for the Hebrew language support, RTL layout, and translation pipeline.

---

## Overview

MEP supports two languages:

| Language | Code | Direction | URL Prefix | Default |
|----------|------|-----------|-----------|---------|
| English | `en` | LTR | `/` (none) | ✅ Yes |
| Hebrew | `he` | RTL | `/he/` | No |

All pages exist in both languages. Hebrew content is translated via Google Gemini during the data pipeline, and UI strings are handled by a static translation dictionary.

---

## Architecture — Two Translation Layers

### Layer 1: UI Strings (Static)

All UI chrome (buttons, labels, headings, disclaimers) is translated via a static dictionary.

| File | Purpose |
|------|---------|
| `src/i18n/ui.ts` | Translation dictionary — all UI strings in `en` and `he` |
| `src/i18n/utils.ts` | Helper functions: `t()`, `getLangFromUrl()`, `isRtl()`, `localizedPath()` |

**Usage in Astro components:**

```astro
---
import { getLangFromUrl, t } from '../i18n/utils';
const lang = getLangFromUrl(Astro.url);
---
<h1>{t(lang, 'dev.keyDevelopments')}</h1>
```

`t(lang, key)` falls back to English if a Hebrew key is missing.

### Layer 2: Content Translation (Dynamic, via Gemini)

Article titles, snippets, sitrep text, developments, and daily briefings are translated from English to Hebrew at pipeline time using Google Gemini. Translations are stored as `_he` suffixed fields alongside the originals.

| File | Purpose |
|------|---------|
| `src/translate-hebrew.js` | Gemini-powered batch translation module |

**Translated content types:**

| Content | English Field | Hebrew Field | When Translated |
|---------|---------------|--------------|-----------------|
| Article title | `title` | `title_he` | Every fetch cycle (Layer 1) |
| Article snippet | `snippet` | `snippet_he` | Every fetch cycle (Layer 1) |
| Sitrep summary | `summary` | `summary_he` | Every synthesis (Layer 2) |
| Sitrep analysis | `detailed_analysis` | `detailed_analysis_he` | Every synthesis (Layer 2) |
| Top update headline | `headline` | `headline_he` | Every synthesis (Layer 2) |
| Development headline | `headline` | `headline_he` | Every fetch cycle (Layer 1) |
| Development summary | `summary` | `summary_he` | Every fetch cycle (Layer 1) |
| Daily briefing | `headline`, `summary`, `outlook`, etc. | `*_he` fields | Daily at 23:55 |

**Hebrew detection**: Articles already in Hebrew (e.g., from Ynet's Hebrew feed) are detected via Unicode range check (`\u0590-\u05FF`) and skip translation — `title_he = title`.

## Translation Module (`src/translate-hebrew.js`)

### Exported Functions

| Function | Purpose |
|----------|---------|
| `initModel(apiKey)` | Initialize Gemini model instance |
| `translateArticles(model, articles)` | Batch-translate article titles & snippets |
| `translateSitrep(model, sitrep)` | Translate situation report fields |
| `translateDevelopments(model, developments)` | Translate key development headlines & summaries |
| `translateDailyBriefing(model, briefing)` | Translate end-of-day briefing |
| `isHebrew(text)` | Detect if text contains Hebrew characters |

### Batch Processing

Articles are translated in batches of 15 to stay within Gemini's token limits. A 1-second delay between batches prevents rate limiting.

### JSON Repair

Gemini sometimes returns broken JSON when translating Hebrew (unescaped quotation marks are common in Hebrew text). The `repairJson()` function handles:
1. Standard `JSON.parse()` attempt
2. Hebrew quotation mark escaping (`"` between Hebrew characters)
3. Array/object extraction from surrounding text
4. Trailing comma cleanup

### Error Handling

If translation fails for a batch or content type, the module falls back to English text — no Hebrew field is ever left undefined. Every article is guaranteed to have `title_he` and `snippet_he` after processing.

## Page Structure

Each page exists twice — English and Hebrew:

```
src/pages/
├── index.astro          # English dashboard
├── videos.astro         # English videos
├── archive.astro        # English archive
├── about.astro          # English about
└── he/
    ├── index.astro      # Hebrew dashboard (RTL)
    ├── videos.astro     # Hebrew videos (RTL)
    ├── archive.astro    # Hebrew archive (RTL)
    └── about.astro      # Hebrew about (RTL)
```

**⚠️ Known duplication**: Hebrew pages are near-copies of their English counterparts with `_he` field references and RTL adjustments. Changes to English pages must be manually mirrored to Hebrew pages. This is a known maintenance risk.

## i18n Helper Functions (`src/i18n/utils.ts`)

| Function | Signature | Description |
|----------|-----------|-------------|
| `t` | `(lang: Lang, key: string) → string` | Look up translated string; falls back to English |
| `getLangFromUrl` | `(url: URL) → Lang` | Detect language from URL path (`/he/...` → `'he'`) |
| `isRtl` | `(lang: Lang) → boolean` | Returns `true` for Hebrew |
| `getLangPrefix` | `(lang: Lang) → string` | `'en'` → `''`, `'he'` → `'/he'` |
| `localizedPath` | `(lang: Lang, path: string) → string` | Build localized URL path |
| `getAlternateLang` | `(lang: Lang) → Lang` | Toggle between `'en'` and `'he'` |
| `getSwitchLangUrl` | `(url: URL, lang: Lang) → string` | Build URL for language switcher |

## RTL Support

Hebrew pages use `dir="rtl"` on the `<html>` element (set in `Base.astro` layout). Key RTL considerations:

- **Navigation arrows flip**: `← Previous` becomes `הקודם →`
- **Flexbox direction**: Automatically handled by `dir="rtl"` — no manual CSS needed for most layouts
- **Text alignment**: CSS `text-align: start` respects direction automatically
- **Language switcher**: Always positioned top-right (English) / top-left (Hebrew) via the header component

## How to Add a UI String

1. Add the English string to `src/i18n/ui.ts` under the `en` object:
   ```ts
   'new.key': 'English text',
   ```

2. Add the Hebrew translation under the `he` object:
   ```ts
   'new.key': 'טקסט בעברית',
   ```

3. Use in Astro components:
   ```astro
   {t(lang, 'new.key')}
   ```

If you skip the Hebrew entry, `t()` will fall back to the English string.

---

## How to Add a New Language

1. Add the language to `src/i18n/ui.ts`:
   ```ts
   export const languages = { en: 'English', he: 'עברית', ar: 'العربية' };
   ```

2. Add all translation keys for the new language in the `ui` object.

3. Create `src/pages/ar/` directory with copies of the English pages, updated to use the new language code.

4. Add translation functions in `src/translate-hebrew.js` (or create a new translation module).

5. Update `isRtl()` in `utils.ts` if the language is RTL.

## Gemini Translation Prompts

Translation prompts specify:
- **Style**: Formal Hebrew news/intelligence style (כתבה חדשותית)
- **Proper nouns**: Use standard Hebrew forms (e.g., Netanyahu → נתניהו)
- **HTML preservation**: `<strong>` tags must pass through untranslated
- **JSON output**: Array format with index matching for batch processing
- **Quotation marks**: Use Unicode gershayim (׳/״) instead of ASCII double-quotes to avoid JSON parsing issues
